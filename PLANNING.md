  ---
  TransNovel - AI 기반 웹소설 번역 플랫폼 기획서

  서비스 개요

  TransNovel은 중국어 웹소설을 한국어로 번역하는 AI 기반 번역 플랫폼입니다. Gemini API를 활용하여 서지정보 기반의 맞춤형 프롬프트 엔지니어링으로 고품질 윤문 번역을 제공합니다.

  ---
  1. 사용자 관리 시스템

  1.1 회원 유형

  | 역할   | 설명                                   | 권한                                  |
  |--------|----------------------------------------|---------------------------------------|
  | 작가   | 원고를 등록하고 번역을 의뢰하는 사용자 | 작품 등록, 번역 요청, 번역본 다운로드 |
  | 윤문가 | 번역된 원고를 검수/수정하는 전문가     | 번역본 검토, 수정, 승인               |
  | 관리자 | 플랫폼 전체 관리                       | 전체 관리 권한                        |

  1.2 인증 기능

  - 이메일/비밀번호 기반 회원가입
  - 소셜 로그인 (Google, Kakao)
  - 이메일 인증
  - 비밀번호 찾기/변경

  1.3 프로필 관리

  - 기본 정보 (이름, 연락처, 프로필 이미지)
  - 역할 정보 및 전문 분야
  - 작업 이력 및 포트폴리오

  ---
  2. 작품 관리 시스템

  2.1 작품 등록 프로세스

  [원문 업로드] → [서지정보 입력] → [번역 설정] → [등록 완료]

  2.2 서지정보 데이터 모델

  | 필드         | 타입     | 필수 | 설명                        |
  |--------------|----------|------|-----------------------------|
  | 작품명       | String   | O    | 한글/원어 작품명            |
  | 제작사       | String   | O    | 출판사명, 판권사            |
  | 연령등급     | Enum     | O    | 전체연령가, 15세, 19세      |
  | 작가정보     | Array    | O    | 역할(글/그림/각색) + 작가명 |
  | 장르         | Array    | O    | 복수 선택 가능              |
  | 연재처       | String   | -    | 플랫폼 + URL                |
  | 제작참여사   | Array    | -    | 스텝 정보                   |
  | 연재상태     | Enum     | O    | 완결/연재중/준비중          |
  | 연재게시일자 | Date     | -    | 1화 업로드 일자             |
  | 회차정보     | Object   | O    | 총 회차, 현재 회차          |
  | 표지         | File/URL | -    | 이미지 첨부                 |
  | 줄거리       | Text     | O    | 작품 시놉시스               |

  2.3 장르 목록

  BL, GL, SF, 감성, 개그, 드라마, 로맨스/순정, 로판, 무협,
  성인, 소년, 슬릴러/공포, 스토리, 스포츠, 썰, 액션, 에피소드,
  역사, 옴니버스, 일본/해외, 일상, 판타지, 학원, 히어로, 기타

  2.4 원고 관리

  - 회차별 원문 업로드 (txt, docx 지원)
  - 원문-번역문 매칭 관리
  - 버전 관리 (원본, 1차 번역, 윤문본)

  3. AI 번역 시스템

  3.1 번역 엔진 구조

  ┌─────────────────────────────────────────────────────────┐
  │                    번역 파이프라인                        │
  ├─────────────────────────────────────────────────────────┤
  │  [원문]                                                  │
  │    ↓                                                    │
  │  [서지정보 기반 프롬프트 생성]                             │
  │    - 장르 특성 반영                                      │
  │    - 연령등급 어휘 수준 조정                              │
  │    - 작품 톤앤매너 설정                                   │
  │    ↓                                                    │
  │  [Gemini API 호출]                                      │
  │    ↓                                                    │
  │  [후처리]                                                │
  │    - 문체 일관성 검토                                    │
  │    - 고유명사 통일                                       │
  │    ↓                                                    │
  │  [번역본 저장]                                           │
  └─────────────────────────────────────────────────────────┘

  3.2 프롬프트 엔지니어링 전략

  서지정보를 활용한 맞춤형 프롬프트 구성:

  [시스템 프롬프트]
  당신은 {장르} 장르의 웹소설 전문 번역가입니다.
  - 대상 연령: {연령등급}
  - 작품 톤: {줄거리 기반 톤 분석}
  - 특수 용어: {장르별 용어집 참조}

  [번역 지침]
  1. 원문의 뉘앙스를 살린 자연스러운 한국어 번역
  2. 캐릭터별 말투 일관성 유지
  3. 문화적 맥락에 맞는 로컬라이제이션

  3.3 용어집 관리

  - 작품별 고유명사 사전 (인명, 지명, 스킬명 등)
  - 장르별 기본 용어집
  - 사용자 커스텀 용어 추가 기능

  5. 기술 스택 제안

  Frontend

  - Framework: Next.js 14 (App Router)
  - UI: Tailwind CSS + shadcn/ui
  - 상태관리: Zustand
  - 폼 관리: React Hook Form + Zod

  Backend

  - Framework: Next.js API Routes / Nest.js
  - Database: PostgreSQL + Prisma ORM
  - 인증: NextAuth.js
  - 파일 저장: AWS S3 / Cloudflare R2

  AI/번역

  - 번역 엔진: Google Gemini API
  - 큐 관리: BullMQ (Redis)
  - 실시간 통신: Socket.io / Server-Sent Events

  7. 개발 우선순위

  Phase 1: MVP (핵심 기능)

  1. 사용자 인증 시스템
  2. 작품 등록 (서지정보 입력)
  3. 원고 업로드 (회차별)
  4. Gemini API 연동 번역
  5. 번역본 다운로드

  Phase 2: 고도화

  1. 실시간 번역 진행 상황
  2. 용어집 관리
  3. 윤문가 워크플로우
  4. 대시보드 통계

  Phase 3: 확장

  1. 배치 번역 (대량 회차)
  2. 번역 품질 비교 도구
  3. 협업 기능
  4. API 비용 최적화
